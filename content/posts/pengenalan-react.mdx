## ğŸ” Apa Itu React?

React adalah **pustaka JavaScript open-source** yang digunakan untuk membangun **user interface** (UI), terutama untuk aplikasi web bersifat **single-page application (SPA)**. React dikembangkan oleh Facebook pada tahun 2013, dan sejak itu... boom! Semua developer front-end seperti tiba-tiba punya poster React di kamar.

React tidak seperti jQuery yang hanya buat DOM manipulasi, atau seperti Angular yang adalah framework all-in-one. React **fokus hanya pada tampilan**, alias bagian **View** di arsitektur MVC. Kalau kamu suka filosofi "kerjakan satu hal dan lakukan itu dengan baik", React adalah teman sekamar yang ideal (dan tidak akan makan cemilan kamu).

## ğŸ§  Kenapa Harus React?

Baik, mari kita jawab pertanyaan penting: kenapa kita buang waktu hidup yang terbatas ini untuk belajar React?

1. **Komponen yang Dapat Digunakan Ulang**  
   UI dipecah menjadi potongan kecil (komponen) yang bisa dipakai berkali-kali. Tombol login? Komponen. Header situs? Komponen. Notifikasi "error kamu sendiri"? Juga komponen.

2. **Virtual DOM**  
   React memperkenalkan *Virtual DOM*, representasi ringan dari DOM asli. Ketika ada perubahan, React membandingkan versi sebelumnya dengan yang baru (*diffing algorithm*) dan hanya mengubah bagian yang perlu. Ini bikin performa meningkat tanpa harus mengutuk kecepatan browser kamu.

3. **Deklaratif**  
   Alih-alih bilang ke browser *bagaimana* melakukan sesuatu, kamu cukup bilang *apa* yang kamu mau. Lebih simpel, lebih bersih, lebih tidak seperti kamu saat disuruh menjelaskan perasaan.

4. **Ekosistem dan Komunitas Besar**  
   Dokumentasi melimpah, jutaan repositori open source, dan ribuan manusia baik hati yang rela menjawab pertanyaan bodoh kamu di Stack Overflow.

### 1. JSX (JavaScript XML)  
JSX adalah sintaks campuran HTML di dalam JavaScript. Kedengarannya absurd, tapi sebenarnya nyaman. Contoh JSX:

```jsx
const element = <h1>Hello, dunia React!</h1>;
```

Dengan JSX, kamu bisa nulis komponen UI seperti nulis HTML biasaâ€”tapi dengan kekuatan JavaScript penuh di baliknya. Seperti punya Hulk pakai kacamata baca.

### 2. Komponen

React berbasis pada **komponen**. Ada dua tipe utama:
- **Function Component** â€“ lebih modern, simpel, dan dipakai bersama Hooks.
- **Class Component** â€“ dulu populer, sekarang masih bisa dipakai, tapi dianggap *boomer*.

Contoh Function Component:

```jsx
function Welcome(props) {
  return <h1>Halo, {props.name}</h1>;
}
```

---

### 3. Props dan State

- **Props**: Data yang dikirim ke komponen dari komponen lain. Sifatnya *read-only*. Kayak surat cinta dari orang lainâ€”bisa dibaca tapi jangan diubah.
  
- **State**: Data internal dari komponen. Bisa diubah, bisa bikin UI berubah sesuai kondisi.

Contoh pakai `useState` di Function Component:

```jsx
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Kamu sudah klik sebanyak {count} kali</p>
      <button onClick={() => setCount(count + 1)}>Klik Aku!</button>
    </div>
  );
}
```

---

### 4. Hooks

Sejak React 16.8, ada fitur baru: **Hooks**. Mereka seperti cheat code untuk membuat function component punya kekuatan class component.

- `useState`: Untuk menyimpan data lokal.
- `useEffect`: Untuk efek samping (API calls, manipulasi DOM, dll).
- `useContext`, `useReducer`, `useRef`, dan teman-temannya yang akan bikin kamu merasa sedang ikut kuliah S2 JavaScript.

## ğŸ§ª Belajar React, Mulai dari Mana?

Berikut roadmap versi â€œmanusia normalâ€ (bukan full-stack alien dari masa depan):

1. Pahami dasar JavaScript dengan baik (kalau belum, React akan terasa seperti mimpi buruk).
2. Pelajari HTML dan CSSâ€”jangan males, itu penting.
3. Kuasai JSX dan konsep komponen.
4. Main-main dengan `useState`, `useEffect`, dan *conditional rendering*.
5. Bangun proyek kecil, misalnya:
   - To-do list
   - Kalkulator
   - Aplikasi catatan harian penuh keluh kesah
6. Pahami routing dengan `react-router-dom`
7. Belajar manajemen state lebih lanjut: Context API, Redux, Zustand, Recoil (pilih racunmu).
8. Masuk ke API handling (pakai `fetch`, `axios`)
9. Terakhir: deploy ke Vercel/Netlify dan pamer ke dunia.
